# Roadmap Format

## 1. Core Infrastructure
**Status:** In Progress

- [x] 1.1 Set up project structure
- [x] 1.2 Configure build system
- [ ] 1.3 Implement logging framework
- [ ] 1.4 Add configuration management

## 2. API Development
**Status:** Not Started

- [ ] 2.1 Design API endpoints
- [ ] 2.2 Implement authentication
- [ ] 2.3 Create data models
- [ ] 2.4 Write API documentation

## 3. Testing & Quality
**Status:** Not Started

- [ ] 3.1 Set up test framework
- [ ] 3.2 Write unit tests
- [ ] 3.3 Add integration tests
- [ ] 3.4 Configure CI/CD pipeline

## Why

Consistent roadmap formatting enables:
- Quick progress tracking across projects
- Clear task ownership and dependencies
- Easy identification of blockers
- Uniform structure for Agent OS to generate and update roadmaps

## Rules

- Use numbered headings for task groups (## 1., ## 2., etc.)
- Add **Status:** line after each group heading (Not Started, In Progress, Completed)
- Number tasks within groups using dot notation (1.1, 1.2, etc.)
- Include checkbox `[ ]` or `[x]` before each task number
- Keep task descriptions concise (one line)
- Add optional context (timeline, dependencies, notes) below task list in each group
- Apply this format to all roadmap.md files in Agent OS projects
